Swagger2–学习笔记(包含Swagger视频)

Lesson1 Swagger2概述

NO1.1 Swagger2简介

Swagger2文档生成工具，是我们写代码的时候，解决前后端交互同步的一个重要工具。

  • Swagger2：

    • 介绍：公司中前后端有时候交互的同步，并不是很理想，通过Swagger2规范，你只需要按照它的规范去定义接口及接口相关的信息；再通过Swagger2衍生出来的一系列项目和工具，就可以做到生成各种格式的接口文档，生成多种语言的客户端和服务端的代码，以及在线接口调试页面等等；
    • 优点：
      • 支持多语言；如，Java、PHP等等；
      • 可以直接运行，达到在线测试API接口；
      • RESTful API文档的在线生成，可以保持API文档和API定义的同步更新；

  • 入门案例：

    • 基本步骤：

      • 创建基于SpringBoot的Maven项目，然后导入Swagger2依赖；

        <dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger2</artifactId>
    <version>2.9.2</version>
</dependency>
<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger-ui</artifactId>
    <version>2.9.2</version>
</dependency>

      • 编写一个controller；

        @RestController
public class MyController {

	@GetMapping("hello")
	public String getMessage(){

		return "hello world";
	}
}

      • 编写一个Swagger2的配置类，配置类中使用注解@EnableSwagger2，该注解的作用就是开启Swagger2；

        @Configuration
@EnableSwagger2  //开启Swagger2
public class SwaggerConf {
}

      • 开启项目跑起来，输入http://localhost:8080/swagger-ui.html，就可以访问到swagger2的文档页面了。

 

 

 

 

Lesson2 Swagger2基础

NO2.1 Swagger2配置

  • Swagger2详细配置：

    • 配置Swagger2的UI界面信息：

      @Configuration
@EnableSwagger2  //开启Swagger2
public class SwaggerConf {
    
    //核心配置信息
    @Bean
    public Docket getDocket(){
        return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo());  //配置Swagger2的ui界面的一些信息
    }
	
	//返回一个ApiInfo对象，该对象中就包含了Swagger2的ui界面的一些配置信息
	private ApiInfo apiInfo(){

		return new ApiInfo(
				"marrycode",  //标题
				"xxgc_hui", //描述信息
				"1.0",  //版本号
				"http://www.baidu.com",  //一个url
				new Contact("作者","url","邮箱"),  //作者信息，需要new一个Contact对象
				"",  //这个可以写空字符串
				"",  //这个可以写空字符串
				new ArrayList());  //这两个可以new一个List即可
	}
}

    • 配置Swagger2的接口扫描：

      @Configuration
@EnableSwagger2  //开启Swagger2
public class SwaggerConf {
    
    //返回一个ApiInfo对象，该对象中就包含了Swagger2的ui界面的一些配置信息
	private ApiInfo apiInfo(){

		return new ApiInfo(
				"marrycode",  //标题
				"xxgc_hui", //描述信息
				"1.0",  //版本号
				"http://www.baidu.com",  //一个url
				new Contact("作者","url","邮箱"),  //作者信息，需要new一个Contact对象
				"",  //这个可以写空字符串
				"",  //这个可以写空字符串
				new ArrayList());  //这两个可以new一个List即可
	}
    
    //核心配置信息
    @Bean
    public Docket getDocket(){
        return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())  //配置Swagger2的ui界面的一些信息
            .select()  //配置接口扫描的前提方法
            .apis(RequestHandlerSelectors   //配置要给哪些类生成接口文档
          				.basePackage("cn.dyf.controller"))
            .paths(PathSelectors.ant("/xxx/xxx"))  //为指定访问路径下的类生成接口文档
            .build();  //配置完成后，构建Docket对象
    }	
}

说明：
一、RequestHandlerSelectors有以下几个参数：
	1.basePackage()：指定要扫描的包；
	2.any()：扫描全部；
	3.none()：不扫描；
	4.withClassAnnotation：扫描类上的注解，参数是一个注解的反射对象；
	5.withMethodAnnotation：扫描方法上的注解。
	
二、PathSelectors参数有以下几个参数：
	1.ant()：扫描指定访问路径；
	2.any()：扫描全部访问路径；
	3.none()：不扫描访问路径；
	4.regex()：扫描符合正则表达式的路径。

    • 配置Swagger2的生效环境：

      @Configuration
@EnableSwagger2  //开启Swagger2
public class SwaggerConf {
    
    //返回一个ApiInfo对象，该对象中就包含了Swagger2的ui界面的一些配置信息
	private ApiInfo apiInfo(){

		return new ApiInfo(
				"marrycode",  //标题
				"xxgc_hui", //描述信息
				"1.0",  //版本号
				"http://www.baidu.com",  //一个url
				new Contact("作者","url","邮箱"),  //作者信息，需要new一个Contact对象
				"",  //这个可以写空字符串
				"",  //这个可以写空字符串
				new ArrayList());  //这两个可以new一个List即可
	}
    
    //核心配置信息
    @Bean
    public Docket getDocket(Environment environment){
    
    	//设置Swagger2生效时的环境
    	Profiles profiles = new Profiles.of(""dev,"test"); 
    
    	//通过environment对象的方法判断当前SpringBoot的环境是哪个环境
    	boolean flag = environment.acceptsProfiles(profiles);
    
        return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())  //配置Swagger2的ui界面的一些信息
            .enable(flag)  //配置是否开启Swagger2，默认是true
            .select()  //配置接口扫描的前提方法
            .apis(RequestHandlerSelectors   //配置要给哪些类生成接口文档
          				.basePackage("cn.dyf.controller"))
            .paths(PathSelectors.ant("/xxx/xxx"))  //为指定访问路径下的类生成接口文档
            .build();  //配置完成后，构建Docket对象
    }	
}

说明：设置是否开启Swagger2，调用enable()方法，该方法默认是true；另外，如果想要根据当前SpringBoot的运行环境来决定Swagger2是否生效，可以使用environment的acceptsProfiles()方法来设置Swagger2生效的环境。

    • 配置Swagger2的分组：

      • 只要创建不同的Docket对象，然后调用groupName()方法即可；
      @Configuration
@EnableSwagger2  //开启Swagger2
public class SwaggerConf {
    
    //核心配置信息
    @Bean
    public Docket getDocket(){
        return new Docket(DocumentationType.SWAGGER_2)
        	.groupName("a"));  //配置Swagger2的组名为a
    }
    
    //核心配置信息
    @Bean
    public Docket getDocket(){
        return new Docket(DocumentationType.SWAGGER_2)
        	.groupName("b"));  //配置Swagger2的组名为b
    }
	
	//核心配置信息
    @Bean
    public Docket getDocket(){
        return new Docket(DocumentationType.SWAGGER_2)
        	.groupName("c"));  //配置Swagger2的组名为c
    }
	
}

 

 

NO2.2 Swagger2注解

  • 常用的Swagger2注解：

    //实体类

@ApiModel("用户实体类")
public class User{

	@ApiModelProperty("用户姓名")
	private String name;
	
	@ApiModelProperty("用户年龄")
	private Integer age;

}

说明：只要调用的Controller方法中有返回User类，那么Swagger生成的文档中相应的Controller方法的返回值就会有中文解释
    
//---------------------------------------
    
@RestController
public class MyController {

    @ApiOperation("获取信息的方法")  
	@GetMapping("hello")
	public String getMessage(@ApiParam("这是xx参数") String name){

		return "hello world";
	}
}    

说明：@ApiOperation注解用于方法上，解释说明该方法的用途；而@ApiParam注解则是说明参数。
